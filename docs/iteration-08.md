# 迭代 8：订单状态管理 + 后台订单管理

> **所属阶段**：第二阶段 — 购物车与订单
> **预估时间**：2-3 天
> **难度**：中等
> **前置依赖**：迭代 7

## 目标

实现订单全生命周期管理

## 交付物

- `lib/order-state-machine.ts`：订单状态机（待支付 -> 已支付 -> 已发货 -> 已完成 / 已取消 / 已退款）
- `app/api/orders/[id]/status/route.ts`：PATCH（状态流转，含状态机校验）
- `app/api/orders/[id]/cancel/route.ts`：POST（取消订单 + 库存回滚）
- `app/admin/orders/page.tsx`：后台订单管理（列表 + 筛选 + 发货操作 + 详情查看）
- 超时自动取消逻辑（定时任务或 API 触发检查）

## 核心学习点

- 状态机模式的设计与实现
- 状态流转的严格校验（防止非法跳转）
- 取消订单时的补偿操作（库存回滚）
- node-cron 定时任务基础

## 验证方式

订单可正常流转所有状态，非法状态跳转被拦截，超时订单自动取消
