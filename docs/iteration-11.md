# 迭代 11：模拟支付系统

> **所属阶段**：第四阶段 — 支付与通知
> **预估时间**：2-3 天
> **难度**：进阶
> **前置依赖**：迭代 8

## 目标

理解支付流程和 Webhook 机制

## 交付物

- `lib/payment.ts`：模拟支付网关（生成支付链接、签名校验、回调处理）
- `app/api/payments/create/route.ts`：创建支付单
- `app/api/webhooks/payment/route.ts`：支付回调（签名验证 + 幂等处理 + 更新订单状态）
- `app/(store)/payment/[orderId]/page.tsx`：支付页面（展示支付信息 + 模拟支付按钮）
- payments 表（记录支付流水）

## 核心学习点

- 支付流程的行业标准设计（创建支付单 -> 跳转支付 -> 异步回调）
- Webhook 签名验证原理
- 幂等性处理（防止重复回调导致重复扣款）
- 支付状态与订单状态的解耦

## 验证方式

模拟完整支付流程，回调正确更新订单状态，重复回调不会重复处理
