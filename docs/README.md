# Next.js 全栈电商平台 — 迭代式开发路线

## 总体说明

- 每个迭代产出一个**可运行、可验证**的增量功能
- 每个迭代标注了**核心学习点**，帮助你聚焦关键知识
- 建议每个迭代完成后做一次自测和代码回顾，再进入下一个
- 迭代之间有依赖关系，请按顺序推进

## 阶段总览

| 阶段 | 迭代范围 | 主题 |
|------|----------|------|
| 第一阶段 | 迭代 1-4 | 商品模块 |
| 第二阶段 | 迭代 5-8 | 购物车与订单 |
| 第三阶段 | 迭代 9-10 | 权限与安全 |
| 第四阶段 | 迭代 11-12 | 支付与通知 |
| 第五阶段 | 迭代 13-14 | 搜索与性能 |
| 第六阶段 | 迭代 15-16 | 工程化与部署 |

## 迭代依赖关系

```
迭代 1 (数据库建模)
  -> 迭代 2 (分类管理)
  -> 迭代 3 (商品API + 上传)
     -> 迭代 4 (商品页面 + 前台)
        -> 迭代 5 (前台注册登录)
           -> 迭代 6 (购物车)
              -> 迭代 7 (地址 + 下单)
                 -> 迭代 8 (订单状态管理)
                    -> 迭代 9 (RBAC权限)
                    -> 迭代 11 (模拟支付)
                       -> 迭代 12 (通知系统)
                    -> 迭代 13 (搜索 + 缓存)
                       -> 迭代 14 (看板 + 性能)
        -> 迭代 10 (认证安全升级，可在迭代5之后任意时间做)
        -> 迭代 15 (日志 + 安全，可在迭代8之后任意时间做)
           -> 迭代 16 (Docker + CI/CD)
```

## 推荐节奏

| 迭代 | 预估时间 | 难度 |
|------|----------|------|
| 1. 数据库建模 | 1-2 天 | 入门 |

| 4. 商品页面 + 前台 | 3-4 天 | 基础 |
| 5. 前台注册登录 | 1-2 天 | 入门 |
| 6. 购物车 | 2-3 天 | 基础 |
| 7. 地址 + 下单 | 3-4 天 | 中等 |
| 8. 订单状态管理 | 2-3 天 | 中等 |
| 9. RBAC 权限 | 3-4 天 | 中等 |
| 10. 认证安全升级 | 2-3 天 | 中等 |

总计约 5-8 周，每天投入 2-3 小时的节奏。

## 技术栈

| 层级 | 技术选型 |
|------|---------|
| 框架 | Next.js 16 (App Router) |
| 语言 | TypeScript |
| UI 组件 | Ant Design 6.x |
| 样式 | Tailwind CSS + Ant Design Token |
| 数据库 | MySQL + mysql2 |
| 认证 | JWT (Access + Refresh Token) |
| 校验 | Zod |
| 文件存储 | 本地 -> 可选 OSS |
| 邮件 | Nodemailer |
| 图表 | ECharts / Recharts |
| 部署 | Docker + GitHub Actions |
