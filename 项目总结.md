## 🗄️ 数据库设计

### 数据表结构

#### 1. 用户表 (users)

```sql
CREATE TABLE users (
  id INT AUTO_INCREMENT PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role VARCHAR(20) DEFAULT 'user',
  status TINYINT DEFAULT 1,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**字段说明**:
- `id`: 主键，自增
- `username`: 用户名（唯一）
- `email`: 邮箱（唯一）
- `password_hash`: 加密后的密码
- `role`: 角色（admin/user）
- `status`: 状态（1=正常，0=禁用）
- `created_at`: 创建时间
- `updated_at`: 更新时间

#### 2. 文章表 (articles)

```sql
CREATE TABLE articles (
  id INT AUTO_INCREMENT PRIMARY KEY,
  title VARCHAR(200) NOT NULL,
  content TEXT,
  author_id INT NOT NULL,
  status TINYINT DEFAULT 1,
  views INT DEFAULT 0,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  FOREIGN KEY (author_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

**字段说明**:
- `id`: 主键，自增
- `title`: 文章标题
- `content`: 文章内容
- `author_id`: 作者 ID（外键）
- `status`: 状态（1=已发布，0=草稿）
- `views`: 浏览量
- `created_at`: 创建时间
- `updated_at`: 更新时间

```bash

```


## 🔍 常见问题

### Q1: 如何重置数据库？

```sql
DROP DATABASE IF EXISTS admin_db;
CREATE DATABASE admin_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

然后运行：`pnpm db:init`

### Q2: 忘记密码怎么办？

可以直接在数据库中修改：

```sql
-- 需要先使用 bcrypt 生成新密码的 hash
UPDATE users 
SET password_hash = '新生成的 hash' 
WHERE username = 'admin';
```

或者重新运行 `pnpm db:init` 会重置为默认密码：`admin123`

### Q3: 如何修改端口？

```bash
pnpm dev --port 3001
```

### Q4: 如何部署到生产环境？

```bash
# 1. 构建
pnpm build

# 2. 启动
pnpm start

# 或使用 PM2
pm2 start npm --name "nextjs-admin" -- start
```
