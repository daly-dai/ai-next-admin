# 📖 后台管理系统使用指南

## 🎯 前期准备清单

### ✅ 需要完成的配置步骤：

#### 1. MySQL 数据库配置

**步骤 1: 确保 MySQL 服务已启动**

在命令行中检查 MySQL 服务状态：
```bash
# Windows
net start | findstr MySQL

# 如果未启动，启动服务：
net start MySQL80  # 服务名可能是 MySQL57, MySQL 等
```

**步骤 2: 创建数据库**

登录 MySQL 并创建数据库：
```sql
-- 登录 MySQL
mysql -u root -p

# 输入密码后，执行：
CREATE DATABASE admin_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

# 验证创建成功
SHOW DATABASES;

# 退出
EXIT;
```

**步骤 3: 配置环境变量文件**

在项目根目录找到 `.env.local` 文件，修改以下内容：

```env
# MySQL 数据库配置
DATABASE_HOST=localhost        # 数据库地址，本地一般用 localhost
DATABASE_PORT=3306            # 数据库端口，默认 3306
DATABASE_USER=root            # 你的 MySQL 用户名
DATABASE_PASSWORD=你的密码     # 你的 MySQL 密码（重要！）
DATABASE_NAME=admin_db        # 数据库名称

# JWT 认证配置
JWT_SECRET=your_jwt_secret_key_change_this_in_production

# 应用配置
NEXT_PUBLIC_APP_NAME=后台管理系统
NEXT_PUBLIC_API_URL=http://localhost:3000/api
```

⚠️ **重要提示**：
- `DATABASE_PASSWORD` 必须修改为你实际的 MySQL 密码
- 如果忘记了 root 密码，可以重置 MySQL 密码

#### 2. 安装项目依赖

如果还没有安装依赖，执行：

```bash
pnpm install
```

#### 3. 初始化数据库

运行初始化脚本，会自动：
- 创建用户表（users）
- 创建文章表（articles）
- 插入默认管理员账户
- 插入测试数据

```bash
pnpm db:init
```

看到以下输出表示成功：

```
🚀 开始初始化数据库...

✅ 数据库连接成功！
✅ 数据库初始化完成！

📝 测试账户信息：
   管理员：username: admin, password: admin123
   普通用户：username: test, password: admin123

🎉 现在可以访问 http://localhost:3000/admin/login 开始使用后台管理系统
```

#### 4. 启动开发服务器

```bash
pnpm dev
```

看到以下输出表示启动成功：

```
  ▲ Next.js 16.x.x
  - Local:        http://localhost:3000
  - Ready in xxxms
```

## 🔧 常见问题解决

### ❌ 问题 1: 数据库连接失败

**错误信息**: `ER_ACCESS_DENIED_ERROR` 或 `数据库连接失败`

**解决方案**:
1. 检查 MySQL 服务是否启动
2. 确认 `.env.local` 中的用户名和密码正确
3. 确认数据库 `admin_db` 已创建

**验证步骤**:
```bash
# 在命令行测试能否连接 MySQL
mysql -u root -p
# 输入密码，如果能登录说明账号密码正确
```

---

### ❌ 问题 2: 找不到模块或导入错误

**错误信息**: `Cannot find module`, `Module not found`

**解决方案**:
```bash
# 删除 node_modules 和锁文件，重新安装
rm -rf node_modules pnpm-lock.yaml  # Mac/Linux
rd /s /q node_modules               # Windows PowerShell
del pnpm-lock.yaml

# 重新安装依赖
pnpm install
```

---

### ❌ 问题 3: 端口被占用

**错误信息**: `Port 3000 is already in use`

**解决方案**:

方法 1: 关闭占用 3000 端口的程序
```bash
# Windows
netstat -ano | findstr :3000
taskkill /PID 进程号 /F

# Mac/Linux
lsof -i :3000
kill -9 进程号
```

方法 2: 使用其他端口
```bash
pnpm dev --port 3001
```

---

### ❌ 问题 4: TypeScript 类型错误

**错误信息**: 各种 TypeScript 类型错误

**解决方案**:
这些通常不影响运行，如果要修复：
```bash
# 检查具体的类型错误
pnpm run lint

# 自动修复部分问题
pnpm run lint:fix
```

## 📝 自定义配置

### 修改数据库配置

编辑 `.env.local` 文件：

```env
# 如果使用不同的数据库名
DATABASE_NAME=my_custom_db

# 如果 MySQL 不是默认端口
DATABASE_PORT=3307

# 如果是远程数据库
DATABASE_HOST=192.168.1.100
```

### 修改应用名称

编辑 `.env.local` 文件：

```env
NEXT_PUBLIC_APP_NAME=我的定制后台
```

重启开发服务器后生效。

## 🎓 学习建议

### 第一步：熟悉项目结构

1. 浏览 `app/` 目录下的所有页面
2. 查看 `lib/` 目录了解数据库连接和认证逻辑
3. 查看 `app/api/` 目录了解 API 接口实现

### 第二步：理解数据流

1. **登录流程**: 
   - 用户在登录页输入账号密码
   - 调用 `/api/auth/login` 接口
   - 验证成功后返回 JWT token
   - token 存储在 localStorage
   - 跳转到后台首页

2. **获取用户列表**:
   - 用户管理页面加载时调用 `/api/users`
   - 携带 token 进行认证
   - 后端从数据库查询数据
   - 返回 JSON 格式数据
   - 前端用表格展示

### 第三步：动手实践

**练习 1**: 修改登录页面的样式
- 找到 `app/admin/login/page.tsx`
- 修改背景颜色、卡片大小等

**练习 2**: 添加新的统计项
- 在 `app/api/stats/route.ts` 添加新的统计查询
- 在 `app/admin/dashboard/page.tsx` 添加新的展示卡片

**练习 3**: 完善文章管理
- 创建文章的 API 接口（类似用户管理）
- 实现文章的添加、编辑、删除功能
