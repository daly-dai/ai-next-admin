# 产品需求文档（PRD）模板

> **用途**：产品/业务方填写本文档，提供给架构（AI）进行需求拆解。
> **填写原则**：宁可多写一句，不可模糊一处。架构无法猜测的细节，最终都会变成返工。

## AI 提取清单（画 Demo 时使用）

> AI 在阶段①将用户提供的任意格式文档规范化时，按以下 9 章逐项提取，缺失信息标 `[?]`，不猜测。

| 章节           | 提取要点                                                                |
| -------------- | ----------------------------------------------------------------------- |
| 一、功能概述   | 功能名称（中文）+ 模块标识（英文）+ 一句话描述 + 所属业务域             |
| 二、目标与边界 | 业务目标（1-3项）+ 不做什么（排除项）                                   |
| 三、数据模型   | 实体字段表 + 枚举定义 + 查询参数                                        |
| 四、接口设计   | 标准 CRUD 勾选 + 非标准接口描述 + 接口偏离说明                          |
| 五、页面与交互 | 页面清单 + 列表页（搜索/表格列/操作列）+ 表单（模式+字段+联动）+ 详情页 |
| 六、业务规则   | 权限角色 + 状态流转 + 业务约束 + 默认行为                               |
| 七、关联关系   | 依赖模块 + 被依赖模块 + 路由与菜单位置                                  |
| 八、设计稿     | 原型图/设计稿链接（有则必录）                                           |
| 九、补充       | 技术约束 + 后续迭代 + 开放问题                                          |

### 4.3 接口偏离说明（如有）
