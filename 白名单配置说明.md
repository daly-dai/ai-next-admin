# 🔐 路由白名单配置指南

## 📋 什么是白名单？

白名单用于配置不需要认证即可访问的公开页面，例如：
- 登录页
- 注册页
- 忘记密码页

## 🗂️ 配置文件位置

**文件路径**: `lib/constants.ts`

这个文件包含了所有公开页面的路径配置。

## ⚙️ 当前白名单配置

```typescript
export const PUBLIC_PATHS = [
  '/admin/login',              // 登录页
  '/admin/register',           // 注册页（待实现）
  '/admin/forgot-password',    // 忘记密码页（待实现）
];
```

## 🎯 如何使用

### 1. 添加新的公开页面

如果你需要添加其他不需要认证的页面，只需在数组中添加路径：

```typescript
export const PUBLIC_PATHS = [
  '/admin/login',
  '/admin/register',
  '/admin/forgot-password',
  '/admin/verify-email',       // 验证邮箱页面（示例）
  '/admin/reset-password',     // 重置密码页面（示例）
];
```

### 2. 支持模糊匹配

白名单支持前缀匹配，例如：

```typescript
'/admin/auth'  // 匹配 /admin/auth/* 的所有子路径
```

### 3. 自动应用

配置后，`app/admin/layout.tsx` 会自动使用 `isPublicPath()` 函数判断是否为公开页面：

```typescript
// 如果是公开页面，直接渲染 children，不使用布局
if (isPublicPath(pathname)) {
  return children;
}
```

## 📝 注意事项

### ⚠️ 安全提示

1. **敏感操作仍需认证**
   - 虽然这些页面是公开的，但涉及数据修改的 API 必须验证身份
   - 例如：注册 API 应该有防刷机制

2. **Rate Limiting**
   - 登录、注册、重置密码等接口应该限制请求频率
   - 防止暴力破解和恶意注册

3. **HTTPS**
   - 生产环境必须使用 HTTPS
   - 防止中间人攻击

### 💡 最佳实践

1. **集中管理白名单**
   - 所有公开路径都在 `lib/constants.ts` 中配置
   - 便于维护和审计

2. **清晰的代码注释**
   - 标注每个路径的用途
   - 标注是否已实现

3. **定期审查**
   - 定期检查白名单中的路径
   - 确保没有误加敏感页面
